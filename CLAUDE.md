## 1. Project Overview

A **digital wedding "Save the Date"** for **Sandro & Kopi**, getting married in
**Syros, Greece on 15 September 2026**. Two pages:

- **`/`** — the invitation: a mobile-first, 3-screen, scroll-snapping experience
  (cover → island guessing game → the day + RSVP).
- **`/details`** — a guest guide (how to get to Syros, ferries, where to stay,
  ceremony, venue).

**Goals:** elegant + playful + interactive; flawless on smartphones released after
~2020 (iOS + Android). It is a **small, targeted guest list** — not a mass-audience
site — so favor simple, robust solutions over heavy infrastructure.

**Live:** https://wedding.khasia.ge (custom domain) — also
https://sandrobetterfly.github.io/w_invitation/
**Repo:** `sandrobetterfly/w_invitation` (GitHub Pages, branch `main`, `/` root).

## 2. Architecture & Tech Stack

**Plain static site. No framework, no build step, no npm, no bundler.**
Everything is hand-written HTML/CSS/JS, all **inline in single files**.

- **HTML/CSS/JS** vanilla. CSS uses custom properties (design tokens) in `:root`.
- **Fonts:** Google Fonts — `Cormorant Garamond` (serif), `Jost` (sans).
- **Icons/illustrations:** hand-drawn **line-art SVG** in an inline `<defs>` sprite
  (`<symbol id="i-...">`), used via `<use href="#i-...">`. No icon library.
- **Confetti:** custom `<canvas>` particle code (no library).
- **RSVP backend:** Google **Apps Script** web app → appends rows to a Google Sheet.
- **Hosting:** GitHub Pages + custom domain.

- `<head>`: meta + OpenGraph. **Per-page share images:** home `/` uses `cover.jpg`

  Home og:title = "Save the Date" + the "find the island…" description. Favicon (inline SVG
  heart), an early `<script>` setting `history.scrollRestoration = 'manual'`.

- 3 `<section class="screen">` (`#s0`, `#s1`, `#s2`).
- RSVP `<div class="rsvp-modal">` + `<canvas class="confetti-canvas">`.
- One big inline `<script>` (config + all behavior).

### ❌ Blocked / broken
**Nothing is broken.** Known *limitations* (accepted given the targeted audience):
- **~2% transient Apps Script failures** under burst. Front-end uses `mode:'no-cors'`
  so it can't detect them (shows thank-you regardless). Mitigations were proposed but
  not built: (a) lighten the script, (b) retry + timestamp de-dupe, (c) swap to a

- **RSVP lock is per-browser** (localStorage) — a different browser / private mode
  could submit again. Acceptable for a no-login site.
- **iPhone SE (375×667)**: screen 3 sits slightly below the fold (nudge-scroll to the
  RSVP button). Modern phones (≥740 tall) fit. We deliberately do NOT special-case
  small/old screens.
- **Tiny islands (~16px)** are fiddly to tap — but Syros (the only one that matters)
  is large.
- Sound output / true iOS audio-unlock can't be verified in the headless preview —
  needs a real iPhone check.

## 5. Code Style & Standards

- **Single self-contained files.** No build, no framework, no external JS deps. New
  features go inline into the relevant `index.html`. Keep it dependency-free.
- **Design tokens first.** Use the `:root` CSS variables for all colors/fonts. The
  accent/CTA color is **`--gold` (coral #FF7653)** despite the name.
- **Mobile-first**, two font weights (400/500), **sentence case** copy.
- **Icons:** add new line-art glyphs as `<symbol>` in the inline sprite; outline
  style, `stroke="currentColor"`, round caps; reference with `<use href="#i-name">`.
- **Animations:** prefer `transform`/`opacity` (GPU-composited). **No
  `background-attachment: fixed`** and no fixed full-screen layers (kills mobile scroll
  perf). Respect `@media (prefers-reduced-motion: reduce)`.
- **Scroll model:** the document is the scroller; `scroll-snap-type` lives on
  `html.snap` and is enabled by JS only after `load`. Don't put `scroll-snap-type`
  back into the static CSS — it reintroduces the "opens on screen 2 after reload" bug.
- **Config constants** live at the very top of the `index.html` `<script>`:
  `GO_LIVE`, `RSVP_DEADLINE` (auto-derived), `RSVP_ENDPOINT` (the Apps Script `/exec`).
- **Clean URLs:** `/details` is `details/index.html` (GitHub Pages serves dir indexes).
- **macOS git gotcha:** `core.ignorecase` is ON. Do NOT use extension patterns like
  `*.JPG` in `.gitignore` — they also match `cover.jpg`. Ignore originals by exact name.
- **Media optimization** (no Homebrew on this machine — use the bundled ffmpeg):
  - Video → 540p H.264, 30 fps, CRF ~25–26, `+faststart`, AAC. Keep source `.MOV`
    gitignored; commit only `wedding.mp4`.
  - Photo → `sips` resize to ~1200px long edge, JPEG q≈72. Keep ~<300 KB (also helps
    the WhatsApp link-preview fetch).
- **Commits:** describe the change; **end the commit message with**
  `Co-Authored-By: Claude <noreply@anthropic.com>` (the harness convention).
  Only commit/push when the change is verified.

## 6. Execution Commands

> No package manager, no build, no test suite. It's static files.

### Local preview (dev server)
`python3 -m http.server` is sandbox-blocked here, so a tiny Node server is used:
```bash
# from the repo root
node .claude/serve.js          # serves on http://localhost:4599 (handles /details dir index)
# then open http://localhost:4599/  and  http://localhost:4599/details
```
Any static server also works, e.g. `python3 -m http.server 8000` (if not sandboxed).

### Build
**None.** The files in the repo *are* the deploy artifact.

### Tests
**None.** Verify visually in the preview + by reading the code. The repo includes a
device/endpoint smoke approach: POST to `RSVP_ENDPOINT` with varied User-Agents via
`curl` to confirm the RSVP pipeline (see session history) — not an automated suite.

### Deploy
```bash
git add <files> && git commit -m "..."   # end msg with Co-Authored-By line
git push origin main                      # GitHub Pages auto-builds (~1 min)
# Live: https://wedding.khasia.ge  (and https://sandrobetterfly.github.io/w_invitation/)
```

### Replace the screen-3 video (transcode + optimize)
```bash
FF=$(python3 -c "import imageio_ffmpeg as f; print(f.get_ffmpeg_exe())")
"$FF" -y -i "SOURCE.MOV" -vf "scale=540:-2,fps=30" \
  -c:v libx264 -crf 26 -preset slow -profile:v high -pix_fmt yuv420p \
  -movflags +faststart -c:a aac -b:a 96k wedding.mp4
# (pip install --user imageio-ffmpeg  if the ffmpeg binary isn't present yet)
```

### Optimize a new cover photo
```bash
sips -Z 1200 -s format jpeg -s formatOptions 72 "SOURCE.JPG" --out cover.jpg
```

### RSVP endpoint (Apps Script)
Source: `rsvp-apps-script.gs`. Deploy from the Google Sheet → Extensions → Apps Script
→ Deploy as Web app ("Anyone" access). To update without changing the URL:
**Deploy → Manage deployments → edit (✏️) → Version: New version → Deploy.**
The resulting `/exec` URL goes into `RSVP_ENDPOINT` in `index.html`.
